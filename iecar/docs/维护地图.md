# 维护地图

## STM32 PlatformIO

| 模块 | 路径 | 维护要点 |
|---|---|---|
| 构建入口 | `stm32f407_platformio/platformio.ini` | `car1`、`car2`、`native` 三个环境 |
| 角色配置 | `src/car1/SYSTEM/CHE/car_project_config.h`, `src/car2/SYSTEM/CHE/car_project_config.h` | 只放角色差异配置 |
| 单源协议生成物 | `src/COMMON/CHE/protocol_contract_generated.h` | 由 `tools/protocol_contract.py` 生成，不手写 |
| 共享默认配置 | `src/COMMON/CHE/car_project_config_shared.h` | 协议开关、超时、速度、阈值默认值 |
| 协议解析核心 | `src/COMMON/CHE/protocol_core_shared.h` | K210/OpenMV RX buffer parser、CarLink frame/parser helper |
| 巡线控制 | `src/COMMON/CHE/car_ctrl_shared.c` | 统一处理 OpenMV no-frame/stale/line/station 事件 |
| OpenMV 接收 | `src/COMMON/CHE/car_uart2_shared.c` | 首帧闸门、版本/校验、站点防抖 |
| K210 接收 | `src/COMMON/CHE/k210it_shared.c` | 首帧、freshness、confidence、position、投票窗口契约 |
| CarLink | `src/COMMON/CHE/ctrl_num_shared.c` | ACK/retry/heartbeat/role/idempotency/compat role normalization |
| 诊断 | `src/COMMON/CHE/car_diag_shared.*` | ring buffer 与高频故障节流 |
| 任务运行时 | `src/COMMON/STRATEGY/task_runtime_common_shared.h` | 统一状态表、状态 trace、完成/安全停车记录 |
| Task1 | `src/COMMON/CHE/task1_shared.c` | 共享任务实现 |
| car1 Task2/3 | `src/car1/SYSTEM/CHE/task2.c`, `task3.c` | car1 主动发送链路，状态表登记 |
| car2 Task2/3 | `src/car2/SYSTEM/CHE/task2.c`, `task3.c` | car2 等待/执行/完成链路，状态表登记 |

## K210 / OpenMV

| 路径 | 维护要点 |
|---|---|
| `tools/protocol_contract.py` | 三端协议单源配置，只在这里改版本/帧常量/阈值 |
| `tools/generate_protocol_contract.py` | 生成 STM32/K210/OpenMV/host 测试常量 |
| `k210代码/protocol_contract.py` | 生成文件，K210 运行时导入 |
| `k210代码/boot.py` | 发送 12 字节带 version byte 的 K210 wire frame |
| `opemv代码/protocol_contract.py` | 生成文件，OpenMV 运行时导入 |
| `opemv代码/巡线.py` | 发送 12 字节带 version byte 的 OpenMV wire frame |
| `docs/protocol_spec.md` | 三端 wire format 与 STM32 RX buffer 规格 |

## 测试/验证

| 路径 | 维护要点 |
|---|---|
| `tools/protocol_golden_tests.py` | 区分 wire frame 与 STM32 RX buffer 的黄金帧测试 |
| `tools/replay_validation.py` | 协议、状态、角色、兼容路径的静态/回放检查 |
| `tools/check_wrapper_boundaries.py` | wrapper 薄转接语义检查 |
| `stm32f407_platformio/test/host/test_protocol_core.c` | host C 测试，共享 parser/strategy 无硬件依赖验证 |
| `stm32f407_platformio/test/native/test_protocol_contract.cpp` | PlatformIO native 测试入口 |

## legacy 边界

- `car1` 的 Task3 仍按角色契约标记为 legacy，不作为默认可用任务。
- `sbus_usart2.c` 当前承载 CarLink USART2 中断初始化，文件名保持兼容；业务协议边界已收敛到 `ctrl_num_shared.c`。
- wrapper include `.c` 仍作为兼容迁移路径保留；`tools/check_wrapper_boundaries.py` 保证 wrapper 只做薄转接。
